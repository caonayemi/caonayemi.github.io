The Position Property

  • Static

    The default value for the position property. Positioned according to the normal flow of the page and unaffected by the top, bottom, left, and right properties.

  • Relative

    Moves from its original (static) position according to the values in the top, bottom, left, and right properties. It can move and overlap other elements, but the original position still functions as part of the flow of elements.

    Common usage: container blocks for absolutely positioned elements.

  • Absolute

    Positioned relative to its first parent element that has a position other than static (If there's no non-static parent, the containing block is <html>) Similar to relative, but within a parent block instead of the full page. Removed from the flow of elements, so other elements behave as though it doesn't exist. Thus, other elements can fall behind the fixed element.

  • Fixed

    Positioned relative to the browser window, as opposed to the page. Doesn't move when page is scrolled. Removed from the flow of elements, so other elements behave as though it doesn't exist. Thus, other elements can fall behind the fixed element.

The Float Property

HTML elements normally stack on top of one another. The float property allows an element to shift horizontally -- using the values right and left -- while elements after the floated element wrap around it if there is room on the page. Elements after a floated element behave as though the floated element is missing from the flow, but do not overlap.

Common usage: images, layout elements.

To prevent an element from wrapping around a floated element, you can appy the clear property to the wrapping element. The values right, left and both indicate which floated element to avoid.
